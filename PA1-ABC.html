Spaceship Momentum

Anka is working at MarsTravel company. She is building a spaceship's module for landing on Mars. For safely landing, the momentum of the spaceship must be rapidly determined every millisecond, with highest precision. Anka knows that, to calculate the spaceship's momentum, she just needs to multiply its velocity to relativistic mass. To avoid any precision loss, these values must be stored in some "radix point" format, with length up to thousands of digits. However, the current spaceship's computer doesn't support that. Therefore, Anka has to re-program the multiplication operation. Help her to implement this in C++ or Java.

Your task

You are given the spacecraft's velocity V and relativistic mass M. Compute its momentum P. These values are non-negative real numbers, written in radix point format in some base-B positional numerical system.

Technical information

A real number in radix point format has at most two parts: an integer part (compulsory) and a fractional part (optional), separated by a radix point (i.e. a small dot "."), if the fractional part is present. It is similar to the fixed point format, except that there's no limit on the length of each part.

Note that, numbers can be written in some base-B numerical system. We consider any radix base B from 2 to 36: B = 2 .. 36. Only following digits are allowed:
"0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ"
Value of digits: 
'0' = 0; '1' = 1; ... '9' = 9; 'A' = 10; 'B' = 11; ..., 'Z' = 35;

Examples of some numbers in different bases: 

In base B=2: 	1010101.01010101; 10010101.01
In base B=10:	9813.1239555; 2015
In base B=16: 	FAFA.00EF
In base B=36: 	CS3230.FUN; JANUARY.2015

Numbers must be written without (unnecessary) leading and trailing zero. All numbers in previous examples are correctly written. Following are examples of incorrectly written numbers: 
Wrong:			Correct:
0089.23		(must delete leading zeros: 	89.23)
1F2.00F510	(must delete trailing zeros: 	1F2.00F51)
69.		(must delete radix point:	69)
.XYZ		(must insert integer part:	0.XYZ)

Hints from the Boss

The Boss gives Anka some hints. First, the long multiplication can be implemented in the same way as how she learned in primary school. Following is an example from Wikipedia. 
	23958233
            5830 x
    ------------
        00000000 ( =      23,958,233 x     0)
       71874699  ( =      23,958,233 x    30)
     191665864   ( =      23,958,233 x   800)
    119791165    ( =      23,958,233 x 5,000)
    ------------
    139676498390 ( = 139,676,498,390        )
Second, such multiplication algorithm works for any radix base. Thus, Anka doesn't need to convert numbers into different bases.

Anka is good enough, so she knows all these stuffs already. She also knows how to store a long number using an array, in reverse order of digits, with the length stored in the first index. This trick her Boss doesn't know. So, she will never ask for more hints.

Input:

First line of the input is the number T of test cases. Next 3*T lines of input describe T test cases. Each test case starts with a number B, the radix base, in the first line. Then follow the velocity V and the mass M, each value per line.

The numbers T and B are in decimal form, while V and M are in base-B form. You can assume that these numbers are given in correct format (no unnecessary zeros and no unnecessary radix point).

Output:

You must output T lines, each line writes the momentum value P for each test case in order given in the input. P must be written in the correct format and in same radix base B indicated in each test case.

Constraints:

Length of V and M is at least 1 and at most 5005 digits.
T is at least 1 and at most 20.
Total length of all numbers in all T testcases is less than 10010 digits.

Memory: 256M
Running time: 1 second per T testcases.

You are required to implement your multiplication algorithm from crash. You are not allowed to use the built-in BigInteger or similar class. Your program will be manually checked.

Always check for the most updated requirements announced in IVLE.
